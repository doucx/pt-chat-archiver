## v7.2.0: 逻辑时序与数据一致性补丁

这是一个专注于“底层逻辑正确性”的关键更新。我们重新审视了消息在时间轴上的排列逻辑，解决了在特定极端情况下（如历史记录回扫与实时录制重叠时）可能出现的顺序颠倒问题，确保您的聊天数据库在任何时候都拥有严丝合缝的逻辑时序。

### 1. 核心突破：序列修复引擎 (Sequence Repair Engine)

*   **逻辑超越物理**：针对从历史记录解析到的时间戳（精度仅到分钟，如 `10:00:00`）可能早于已记录的实时消息（精度到毫秒，如 `10:00:32`）的问题，我们引入了“序列修复”算法。
*   **智能 ID 重写**：当系统检测到逻辑上在后的消息因时间精度问题导致 ID 偏小时，会自动基于前一条消息的时间戳推断出正确的“逻辑位置”，并为其重写 ULID。这确保了在存储引擎（IndexedDB）中，消息永远按照它们在屏幕上出现的真实顺序排列。

### 2. 批次解析单调性保证

*   **毫秒级微调**：在处理大批量的历史消息回扫时，系统现在会自动为同一分钟内的每条消息分配微小的毫秒偏移（+1ms）。
*   **消除时间撞车**：即使 DOM 界面上多条消息显示的时间完全一致，它们在数据库中也会拥有严格递增的唯一标识，彻底杜绝了加载历史记录时可能出现的随机乱序。

### 3. 数据层与表现层解耦

*   **所见即所得**：我们的修复逻辑仅作用于用于排序的 ID，而保留了原始捕获到的显示时间。这意味着您的聊天记录既能保证排序绝对正确，又能忠实记录下它在界面上最初显示的样子。

## v7.4.0: 迈向生产级的数据库架构

这是一个具有里程碑意义的重构版本。我们彻底告别了“将数据库当成 JSON 文件读写”的旧模式，完成了向真正意义上的“增量数据库架构”的华丽转型。这不仅消除了大内存占用导致的卡顿，也为数据的绝对安全性提供了坚实保障。

### 1. 核心架构演进：真正的增量存储

*   **告别全量写入**：以前每产生一条消息，系统都会将数万条历史记录重新序列化并写入磁盘。现在，新消息通过增量接口（`putMessage`）直接插入数据库。这彻底消除了长期运行后的系统掉帧和卡顿。
*   **内存零压力**：我们移除了常驻内存的 `inMemoryChatState`。现在系统仅在需要显示时才从数据库调取当前页的数据，即使您的存档高达 100MB，插件的内存占用也将始终保持在极低水平。
*   **数据库索引升级 (V2)**：为了支持秒级的海量数据检索，我们升级了 IndexedDB 架构，引入了 `server_channel_time` 复合索引，大幅提升了翻页和过滤的响应速度。

### 2. 智能对齐算法 (Greedy Alignment Detection)

*   **更强健的查重合并**：弃用了简单的“头部重叠”对比，引入了全新的贪心对齐算法。
*   **容忍数据空洞**：当您因为网络波动或切页导致聊天记录出现局部缺失时，新算法能精准识别出哪些消息是已经存过的，哪些是漏掉需要补上的，从而实现“无损缝合”。

### 3. UI 交互革命：响应式数据驱动

*   **异步渲染引擎**：UI 层与数据层完全解耦。现在的界面通过 `DataAdapter` 异步获取数据，这意味着在后台进行大规模数据处理时，您的操作界面依然丝滑顺畅。
*   **状态单一真相来源**：重构了 UI 状态管理，将当前选中的服务器和频道完全纳入 `uiState` 统一驱动，彻底杜绝了因频繁切换导致的界面显示紊乱（“脏读”问题）。
*   **移除冗余控制**：由于实现了实时增量保存，我们移除了已不再需要的“立即保存”按钮和“自动保存间隔”提示，让界面回归最纯粹的阅读体验。

### 4. 维护工具增强：异步重复项扫描

*   **不再冻结界面**：重写了重复记录清理工具。现在扫描过程（`scanAllDuplicatesAsync`）采用分片异步执行，在扫描数万条记录的同时，您依然可以正常聊天或翻看历史，绝不卡顿。
*   **更精准的清理规则**：优化了爆发期重复项的识别逻辑，能够更智能地识别出因系统故障产生的重复冗余，而不会误删正常的消息。

### 5. 开发者体验 (DX)

*   **全量单元测试覆盖**：针对核心的合并逻辑和 ID 单调性修复逻辑编写了详尽的测试用例，确保在各种极端数据条件下系统都能稳定运行。
*   **健壮性补丁**：修复了在处理缺失时间戳的异常数据时可能导致的日期运算崩溃问题。

## v7.4.2: 逻辑缝合与 ID 零冗余补丁

这是一个专注于“数据绝对纯净”的质量更新。我们重写了核心的合并对齐算法，彻底解决了在频繁开关聊天框时可能出现的旧消息随机“穿越”到最新的灵异现象，并从根源上杜绝了数据库产生冗余记录的可能性。

### 1. 核心算法进化：拉链式精准缝合 (Zipper Merge)

*   **从“追加”到“织入”**：放弃了以往将缺失消息简单追加到末尾的逻辑。新算法像拉链一样精准对齐 DOM 扫描结果与数据库现有记录，一旦发现中间存在数据空洞（例如因切页遗漏的一条消息），会将其**原地缝合**进正确的时间线位置。
*   **消除“随机插入”幻觉**：修复了在对齐失败时，算法错误地将旧消息判定为新消息并插在最前方的 Bug。现在，即便历史记录解析出现微小偏差，系统也能通过深度探查保持整体序列的稳健。

### 2. 彻底根除重复：ID 绝对不可变性

*   **废除级联重写**：我们移除了曾用于修正顺序的“全局 ID 重写”机制。现在，**任何已进入数据库的消息其 ID 和时间戳都将永久固定，绝对不再被修改。** 
*   **免疫存储冗余**：由于 IndexedDB 是基于 ID 存储的，ID 不再变动意味着系统永远不会因为“改名”而产生重复的残留记录。这不仅保护了数据的唯一性，也进一步减缓了存储空间的增长。
*   **局部时间插值**：对于新发现并需要补齐的消息，系统现在采用“局部插值”技术——仅基于其前一条基准消息的时间戳微调 1 毫秒。这种方法既保证了插入后的物理排序正确，又不会产生牵一发而动全身的连锁反应。

### 3. 容错与防御能力提升

*   **50 倍探查视野**：将对齐算法的深度探查窗口从 5 条扩大到了 **50 条**。这意味着即使您的聊天框因为网络极度延迟堆积了大量未记录消息，系统也能游刃有余地完成对齐，不会触发断层误判。
*   **自动化质量防线**：新增了针对拉链合并算法的专项单元测试用例，在模拟各种复杂的断层、重叠和时间戳撞车场景下，确保合并结果的数学准确性。

### 4. 细节优化

*   **日志系统精简**：移除了合并过程中冗余的控制台调试输出，仅在真正发生不可恢复的断层时才会发出警告，使控制台环境更加整洁。
*   **性能优化**: 优化大规模记录下的分页与视图切换性能。

## v7.5.0: 跨端适配与初始化稳定性增强

**连接不同性能的设备。** 这是一个专注于“兼容性与健壮性”的优化版本。我们通过引入可配置的初始化逻辑与容量断路器，彻底解决了由于设备性能差异（尤其是手机端与电脑端）导致的进入游戏时消息抓取不全的问题。

### 1. 动态初始化防抖配置 (Customizable Init Delay)

*   **设备性能适配**：针对手机端 DOM 渲染较慢，可能导致原硬编码的 150ms 防抖过早失效的问题，我们开放了“初始化防抖延迟”设置（50ms - 5000ms）。
*   **消除盲区**：性能较弱的用户可以通过调高该延迟（例如 1000ms 或 1500ms），确保脚本在进入游戏后能稳健地捕获到完整的 100 条历史记录，不再因设备卡顿而丢失开场对话。

### 2. 初始化死循环保护 (Initialization Breaker)

*   **容量断路器**：为防止在高防抖值（如 1500ms）下，由于频道消息过于频繁导致初始化逻辑被不断重置、陷入无限循环的风险，我们引入了“节点计数拦截”机制。
*   **智能终止**：一旦检测到当前加载的消息数量接近历史上限（约 100 条），系统会自动触发断路保护，强制结束初始化阶段并转入实时监听。这保证了即便在极度活跃的频道，脚本也能在预定时间内完成初始化。

### 3. 实时性与一致性优化

*   **监听无缝衔接**：重构了 `activateLogger` 的启动时序。现在的实时监听器会与历史回扫逻辑并行运作，利用 v7.4.2 引入的“拉链式缝合”算法，在毫秒级自动解决快照与实时流之间的重叠查重，确保数据采集不留缝隙。

### 4. 界面改进

*   **配置项扩展**：在设置页面新增了“初始化防抖延迟”输入框，并附带了详细的说明文字，引导用户根据自己的设备性能进行微调。
*   **状态反馈优化**：进一步精简了初始化过程中的内部逻辑，减少了不必要的 CPU 消耗，使脚本在低功耗设备上运行更加轻快。

*   **增强掌控感**：用户现在可以直观地确认脚本当前正在监听哪个具体的聊天窗口。这在处理多路私聊或切换队伍频道时，提供了极佳的心理预期和状态确认。

### 3. 断层警告标记的精准缝合 (Precise Gap Warning Placement)

*   **同步插值逻辑**：现在，系统生成的“断层警告”将作为元数据被正确织入“拉链式缝合”算法。它将获得基于前序记录的 `+1ms` 微观时间戳，确保其在数组顺序与 UI 视觉排序上，都能精准地出现在数据丢失的那个断层点。

## v7.5.3: 丝滑导航与响应式性能补丁

**快如闪电，操作随心。** 这是一个专注于“交互感知与性能极限”的优化版本。我们通过引入元数据缓存与非阻塞渲染架构，彻底消除了在大数据量下翻页和切换视图时的“粘滞感”。

### 1. 毫秒级元数据缓存 (Metadata Caching)
*   **瞬时频道切换**：通过在内存中维护频道消息计数，我们将切换频道时的数据库查询开销从 $O(N)$ 降至 $O(1)$。现在，无论您拥有多少个频道，切换显示都将在瞬间完成。
*   **智能同步**：缓存系统会自动感应消息的增删并实时更新计数，确保性能提升的同时数据绝对准确。

### 2. 非阻塞 UI 渲染 (Non-blocking Rendering)
*   **告别界面冻结**：重构了渲染管线，引入了“主动让出”机制。在加载海量数据（如点击数据统计）时，界面会立即反馈加载状态并保持操作响应，彻底杜绝了以往点击后“卡死”数秒的现象。
*   **竞态控制**：新增渲染 ID 校验机制。即使您疯狂连点“下一页”，系统也会智能丢弃过期的加载请求，仅渲染您最后停留的页面，避免资源浪费与界面闪烁。

### 3. 乐观分页更新 (Optimistic UI)
*   **即时操作反馈**：现在点击翻页按钮时，顶部的页码指示器（如 `15 / 27`）会立即跳动。这种“先反馈、后加载”的策略消除了操作后的心理等待时间，让导航体验更符合直觉。

### 4. 性能稳定性补丁
*   **大数据量兼容**：优化了在大页码（PageSize 达 1000+）下的内存拼接逻辑，显著降低了浏览器在大文本渲染时的 CPU 峰值占用。
