## v6.0.0

这是一个重大的里程碑版本。我们彻底重构了底层存储逻辑，正式开启“多服务器存档”时代。同时，通过对 UI 模块的深度拆解，大幅提升了系统的响应速度与未来的扩展性。

### 1. 核心突破：多服务器独立存档 (V6 架构)

*   **服务器隔离**：不再将所有聊天记录混在一起。新版本会自动识别您当前所在的服务器（如 Safe Server、18+ Server 等），并将聊天记录存入对应的独立空间。
*   **智能切换与查看**：在设置界面，您可以随时切换“查看”不同服务器的历史存档，而这不会干扰当前服务器的数据录制。
*   **状态感知**：顶部标题栏新增了实时状态显示，能明确告知您当前是在“正在记录”还是在查看其他服务器的“只读模式”。

### 2. 平滑的数据迁移系统

*   **自动升级**：检测到旧版本的 V5 存档时，系统会主动提示并帮助您将其迁移至指定的 V6 服务器节点下，确保您的历史数据不丢失。

### 3. 细节优化

*   **一键回正**：在设置中查看其他存档时，点击“📍”按钮即可快速回到当前正在录制数据的服务器视图。

## v6.1.0

建立专门的迁移处理器以支持从 v4 到 v6 的平滑升级，并为未来版本留出扩展空间

## v6.2.0

**回归记录本质。** 本次更新通过极简主义的界面重构，将更多的屏幕空间留给对话内容，同时为追求极致安全的用户提供了全方位的存档控制能力。

### 1. 极简主义交互重构 (Compact UI)

*   **符号化操作**：主界面的文字按钮（统计、设置、复制、关闭）现已全部替换为直观的 Emoji 符号。这不仅让界面更加紧凑现代，也为窄屏用户腾出了宝贵的横向空间。
*   **沉浸式 Header**：我们从主界面移除了版本号、脚本名称和实时录制状态等元信息，将它们收纳进“设置”页面。现在，主界面将仅保留最核心的频道切换与控制，让您的视野更加纯粹。

### 2. 存档确定性与安全增强

*   **保存进度透明化**：在设置页面新增“上次保存时间”显示。您可以实时确认脚本是否成功将最近的对话存入硬盘，消除对数据丢失的焦虑。
*   **手动保存控制**：新增“💾 立即保存”按钮。在经历了一段极其重要的对话后，您可以随时手动触发持久化，而无需等待系统定时器。
*   **自定义保存频率**：开放了自动保存间隔设置。无论您是追求极端安全的“5秒间隔”爱好者，还是希望减少浏览器负担的“长间隔”用户，现在都可以自由定义。

### 3. 维护操作区整合

*   **管理与记录分离**：将“下载备份”功能移入设置页面的维护操作区。这符合“高频操作在表面，管理操作在深层”的逻辑，使日常使用体验更加顺滑。
*   **下载引擎修复**：深度重构了下载逻辑，修复了在 V6 多服务器架构下导出功能失效的问题，现在您可以完美导出所有服务器的完整 JSON 与文本存档。

### 4. 架构微调

*   **UI 模块化提升**：对前端模板进行了组件化位置优化，进一步提升了 UI 的加载效率与未来功能的扩展潜力。

## v7.0.0

**冲破存储极限，阅读体验焕然一新。**

这是一个里程碑式的版本。通过升级底层的“存储引擎”，我们彻底解决了以往存储空间不足导致的报警问题，并为您带来了更智能的自动滚动与阅读功能。

### 1. 存储空间无限扩容

- **告别 5MB 限制**：旧版本经常遇到的“存储空间即将存满”警告已成为历史。新版本支持存储海量聊天记录，无论您有多少服务器存档，都能轻松容纳。
- **空间管理更透明**：设置页面现在会实时显示准确的消息总数以及预估的磁盘占用情况，让您对自己的“数据库”了如指掌。
- **旧数据清理**：在升级到新版后，您可以使用新增的“删除旧版备份”功能，一键清除浏览器中残留的过期缓存，释放系统空间。

### 2. 智能“锁定底部”与自动翻页

- **强力跟随模式**：点击 `>>` 按钮即可开启“锁定底部”模式（按钮变绿）。此时系统会自动跟随最新消息，并支持自动翻页，无需手动干预。
- **丝滑交互体验**：当您手动滚动到页面最底部时，系统会自动锁定；而当您向上翻阅历史消息时，锁定会自动解除，完全符合直觉。

### 3. 只读模式与界面优化

- **沉浸式只读提醒**：我们移除了顶部显眼的警告横幅，改为低调的背景水印（"ARCHIVE READ-ONLY"）。这让您在查看历史存档时，视野更加纯粹。
- **服务器自动跟随**：新增“自动跟随服务器”选项（默认开启）。当您在不同服务器间切换时，界面会自动跳转到对应服务器的记录，无需手动切换。
- **移动端布局修复**：修复了在手机等小屏幕上按钮可能超出屏幕边缘的问题，确保操作依然顺手。

### 4. 数据安全保障

- **无感数据迁移**：首次启动新版本时，系统会自动为您迁移旧版本的所有数据，过程完全自动化，确保您的历史记录万无一失。

## v7.1.0: 数据恢复与安全补丁

这个版本主要修复了 v7.0.0 升级过程中可能导致的数据迁移不全问题，并提供了手动找回历史记录的工具。

### 1. 启动合并提醒
*   **主动询问**：如果您有未迁移的旧版数据，进入游戏时会弹出窗口，询问是否将其合并到当前存档。
*   **安全取消**：如果您暂时不确定，点击“取消”不会丢失任何数据，系统会保留旧数据供以后处理。

### 2. 设置页新增“恢复中心”
如果您在启动时跳过了合并，现在可以在 **[设置 ⚙️]** 页面中找到 **“发现残留数据！”** 区域：
*   **尝试合并**：手动触发旧数据合并，系统会自动排重并将旧记录接在现有历史之前。
*   **放弃并清理**：如果您确认不再需要旧版数据，可以一键删除 localStorage 中的残留，释放存储空间。

### 3. 数据安全改进
*   **防止覆盖**：修复了在特定情况下旧版数据可能会覆盖（而非合并）新版 v7 数据的问题。现在系统会优先保护您现有的记录。

## v7.2.0: 逻辑时序与数据一致性补丁

这是一个专注于“底层逻辑正确性”的关键更新。我们重新审视了消息在时间轴上的排列逻辑，解决了在特定极端情况下（如历史记录回扫与实时录制重叠时）可能出现的顺序颠倒问题，确保您的聊天数据库在任何时候都拥有严丝合缝的逻辑时序。

### 1. 核心突破：序列修复引擎 (Sequence Repair Engine)

*   **逻辑超越物理**：针对从历史记录解析到的时间戳（精度仅到分钟，如 `10:00:00`）可能早于已记录的实时消息（精度到毫秒，如 `10:00:32`）的问题，我们引入了“序列修复”算法。
*   **智能 ID 重写**：当系统检测到逻辑上在后的消息因时间精度问题导致 ID 偏小时，会自动基于前一条消息的时间戳推断出正确的“逻辑位置”，并为其重写 ULID。这确保了在存储引擎（IndexedDB）中，消息永远按照它们在屏幕上出现的真实顺序排列。

### 2. 批次解析单调性保证

*   **毫秒级微调**：在处理大批量的历史消息回扫时，系统现在会自动为同一分钟内的每条消息分配微小的毫秒偏移（+1ms）。
*   **消除时间撞车**：即使 DOM 界面上多条消息显示的时间完全一致，它们在数据库中也会拥有严格递增的唯一标识，彻底杜绝了加载历史记录时可能出现的随机乱序。

### 3. 数据层与表现层解耦

*   **所见即所得**：我们的修复逻辑仅作用于用于排序的 ID，而保留了原始捕获到的显示时间。这意味着您的聊天记录既能保证排序绝对正确，又能忠实记录下它在界面上最初显示的样子。

## v7.4.0: 迈向生产级的数据库架构

这是一个具有里程碑意义的重构版本。我们彻底告别了“将数据库当成 JSON 文件读写”的旧模式，完成了向真正意义上的“增量数据库架构”的华丽转型。这不仅消除了大内存占用导致的卡顿，也为数据的绝对安全性提供了坚实保障。

### 1. 核心架构演进：真正的增量存储

*   **告别全量写入**：以前每产生一条消息，系统都会将数万条历史记录重新序列化并写入磁盘。现在，新消息通过增量接口（`putMessage`）直接插入数据库。这彻底消除了长期运行后的系统掉帧和卡顿。
*   **内存零压力**：我们移除了常驻内存的 `inMemoryChatState`。现在系统仅在需要显示时才从数据库调取当前页的数据，即使您的存档高达 100MB，插件的内存占用也将始终保持在极低水平。
*   **数据库索引升级 (V2)**：为了支持秒级的海量数据检索，我们升级了 IndexedDB 架构，引入了 `server_channel_time` 复合索引，大幅提升了翻页和过滤的响应速度。

### 2. 智能对齐算法 (Greedy Alignment Detection)

*   **更强健的查重合并**：弃用了简单的“头部重叠”对比，引入了全新的贪心对齐算法。
*   **容忍数据空洞**：当您因为网络波动或切页导致聊天记录出现局部缺失时，新算法能精准识别出哪些消息是已经存过的，哪些是漏掉需要补上的，从而实现“无损缝合”。

### 3. UI 交互革命：响应式数据驱动

*   **异步渲染引擎**：UI 层与数据层完全解耦。现在的界面通过 `DataAdapter` 异步获取数据，这意味着在后台进行大规模数据处理时，您的操作界面依然丝滑顺畅。
*   **状态单一真相来源**：重构了 UI 状态管理，将当前选中的服务器和频道完全纳入 `uiState` 统一驱动，彻底杜绝了因频繁切换导致的界面显示紊乱（“脏读”问题）。
*   **移除冗余控制**：由于实现了实时增量保存，我们移除了已不再需要的“立即保存”按钮和“自动保存间隔”提示，让界面回归最纯粹的阅读体验。

### 4. 维护工具增强：异步重复项扫描

*   **不再冻结界面**：重写了重复记录清理工具。现在扫描过程（`scanAllDuplicatesAsync`）采用分片异步执行，在扫描数万条记录的同时，您依然可以正常聊天或翻看历史，绝不卡顿。
*   **更精准的清理规则**：优化了爆发期重复项的识别逻辑，能够更智能地识别出因系统故障产生的重复冗余，而不会误删正常的消息。

### 5. 开发者体验 (DX)

*   **全量单元测试覆盖**：针对核心的合并逻辑和 ID 单调性修复逻辑编写了详尽的测试用例，确保在各种极端数据条件下系统都能稳定运行。
*   **健壮性补丁**：修复了在处理缺失时间戳的异常数据时可能导致的日期运算崩溃问题。
